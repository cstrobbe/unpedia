  • Flanagan, David; Matsumoto, Yukihiro: The Ruby Programming Language. O'Reilly Media, 2008. (448 pages) ISBN 978-0596516178.

  • Shaughnessy, Pat: Ruby Under a Microscope: An Illustrated Guide to Ruby Internals. No Starch Press, 2013. (360 pages) ISBN 978-1-59327-527-3.
  • Ruby-Doc.org: Documenting the Ruby Language: API documentation maintained by the Ruby community.
  • Yegge, Steve: A Quick Tour of Ruby , Stevey's Drunken Blog Rants™, 26.10.2004.
  • Why's (Poignant) Guide to Ruby. This is an introductory book to Ruby that is distributed under a Creative Commons licence. The source code is maintained on GitHub. (There is even a Wikipedia article about why's (poignant) Guide to Ruby.)
  • RubyDoc.info: Documenting RubyGems, Stdlib, and GitHub Projects: website that relies on YARD to automatically generate documentation for open source Ruby code.

Editors and IDEs

  • Arcadia: a minimalistic Editor for Ruby written in Ruby and developed with itself. This is an open source editor available on GitHub under the terms of the Ruby licence. The source code was last updated in late 2015.
  • JetBrains: RubyMine: a commercial editor; a 30-day trial can be downloaded for free.
  • EasyEclipse: EasyEclipse for Ruby and Rails. This is an open source tool licensed under the terms of the Open Software License version 2.1. It is not clear whether this package has been updated since 2011.
  • Richardson Software: EditRocket: a commercial source code editor for Microsoft Windows, Mac OSX and GNU Linux that includes Ruby-specific features and tools.
  • Redcar: an open source editor written in Ruby that runs on JRuby. The code is available on GitHub under the conditions of the GPL 2.0 licence. The website points out that the editor is not finished; the code has not been updated since 2013.

Beginning Ruby programmers frequently ask what editor or IDE is the best choice, or what tools other Ruby programmers use. The most common recommendations are for general editors and IDEs rather than dedicated Ruby tools. See for example the following articles and discussions.

  • Abder-Rahman Ali: What Editor Do Rubyists Use? SitePoint, 22.12.2014.
    The author asked 100 Ruby programmers what editors they use, which resulted in the following list of editors (listed in order of preference): vim, Sublime Text, Emacs, Atom, TextMate, Sakura, Pico. Yukihiro Matusmoto (Matz), who invented Ruby, preferred Emacs at that time.
  • What Ruby IDE do you prefer? (A question on StackOverflow, originally posted in 2009.)

Ruby Libraries and Tools

  • The Ruby Toolbox: a catalogue of Ruby software. The source code is maintained on GitHub.
  • Awesome Ruby: A collection of awesome Ruby gems, tools, frameworks and software, maintained as a community effort.
  • IronRuby: an open-source implementation of the Ruby programming language which is tightly integrated with the .NET Framework.
  • YARD - A Ruby Documentation Tool.
  • JRuby: an open source implementation of Ruby that runs on the Java Virtual Machine. See also JRuby on Wikipedia and Tim Bray's blog post about the release (14.06.2007).

Other Ruby Links

  • Atwood, Jeff: Why Ruby?, Coding Horror, 22.03.2013.
    In this blog post, Jeff Atwood explains why he chose Ruby instead of .NET for building Discourse.
  • Yegge, Steve: Tour de Babel, Stevey's Drunken Blog Rants™, September 2004.
    This blog post is not specifically about Ruby, but Yegge writes:

    For the most part, Ruby took Perl's string processing and Unix integration as-is, meaning the syntax is identical, and so right there, before anything else happens, you already have the Best of Perl. And that's a great start, especially if you don't take the Rest of Perl.

    But then Matz took the best of list processing from Lisp, and the best of OO from Smalltalk and other languages, and the best of iterators from CLU, and pretty much the best of everything from everyone.

    And he somehow made it all work together so well that you don't even notice that it has all that stuff. I learned Ruby faster than any other language, out of maybe 30 or 40 total; it took me about 3 days before I was more comfortable using Ruby than I was in Perl, after eight years of Perl hacking. It's so consistent that you start being able to guess how things will work, and you're right most of the time. It's beautiful. And fun. And practical.

We release a conversation with notable programmers and Rubyists each week to help programmers advance in their careers and skills.
