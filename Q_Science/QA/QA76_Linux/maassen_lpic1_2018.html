What is this page about?

This page contains notes and comments related to the book LPIC-1. Sicher zur erfolgreichen Linux-Zertifizierung by Harald Maaßen (Rheinwerk, 2018). This was probably the first LPIC-1 preparation guide that was updated to version 5 of the LPIC-1 exam objectives. The book's author has been a member of the Linux Professional Institute (LPI) since 2014. See also his blog post Recent Technical Innovation Requires New Exams: LPIC-1 version 5.0 (15 January 2019).

Even though Harald Maaßen is a member of LPI, some of the information in his book is incomplete or not completely accurate. This page lists notes and comments on this type of issues.
For references, see the objectives for LPIC-1 Exam 101 Objectives and LPIC-1 Exam 102 Objectives

Topic 101: System Architecture

101.1: Determine and configure hardware settings

Page 29: /proc: see also /proc (TLDP, no date), PCI Drivers (Linux Device Drivers, third edition, by Greg Kroah-Hartman, Alessandro Rubini and Jonathan Corbet; O'Reilly, 2005), /proc/interrupts, /proc/pci, /proc/scsi/, /proc/cpuinfo and /proc/iomem in the Red Hat Enterprise Linux documentation.

Note that the book does not explain what I/O ports, interrupts and DMA are; nor does it list important IRQs and which types of hardware use them. Below is a list of IRQs based on the YouTube video LPI101 101 1 Teil2 Deutsch (January 2017), the article Introduction to IRQs, DMAs and Base Addresses by Eugene Blanchard (1999) and the wiki page IRQ, ACPI and APIC and the Linux kernel by Niccolo Rigacci.

IRQFunction
IRQ 0 System Timer
IRQ 1 Keyboard Controller
IRQ 3 Com2 (default) / Com4
IRQ 4 Com1 (default) / Com3
IRQ 5 Sound card or LPT2
IRQ 7 LPT1 (Parallel port) or sound card
IRQ 14 Primary IDE controller
IRQ 15 Secondary IDE controller

Page 30: sysfs and /sys: see also Understanding The sysfs File System (/sys) in Linux (The Geek Diary, no date), which explains the purpose of the contents of /sys:

  • /sys/block: This directory contains entries for each block device in the system;
  • /sys/bus: This directory contains subdirectories for each physical bus type supported in the kernel;
  • /sys/class: This directory contains every device class registered with the kernel;
  • /sys/devices: This directory contains the global device hierarchy of all devices on the system;
  • /sys/firmware: This directory contains subdirectories with firmware objects and attributes;
  • /sys/module: This directory contains subdirectories for each module that is loaded into the kernel;
  • /sys/power: The system power state can be controlled from this directory.

Page 31: see also udev - Linux dynamic device management in the Debian wiki and Tutorial on how to write basic udev rules in Linux by Egidio Docile (02.12.2017).

103.3: Perform basic file management

Page 103: gzip: in order to keep the file that is being compressed, you need the option -k or --keep (not -c). In addition, you don't need to redirect the output using >, since there is no point in outputting the compressed version of a file to standard output. It is sufficient to run, for example, gzip log.txt or (if you want to keep the original file) gzip -k log.txt.

Page 103: gzip: the book points out that you can pass a list of file or a glob pattern to compress multiple files in a single step. One should be aware that this creates a gz file for each file that matches these parameters.
